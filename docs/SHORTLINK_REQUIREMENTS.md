# 短链接需求

状态：v2.0 已实现

## 项目目标

HaoShortLink 在通用 `HttpServer/` 框架之上提供一个可直接使用、可观察、可恢复并可重复验证的短链接服务。

最初的 V1 最小闭环仍是底线：

```text
服务可启动 -> 创建短链 -> 匿名访问短码 -> 返回 302
```

v2.0 在该闭环上增加真实用户入口和对象权限，但不改变公共跳转的匿名语义。

## v2.0 功能需求

### 身份与会话

- 用户可使用唯一用户名和密码注册、登录、退出并查询当前身份。
- 用户名大小写不构成不同账号；密码只保存专用哈希。
- 会话由服务端持久化，客户端使用 `HttpOnly` Cookie，数据库只保存 token 摘要。
- 会话固定过期、可显式撤销；禁用账号的旧会话立即视为无效。
- 登录失败不能区分账号不存在、密码错误或账号禁用。
- 浏览器状态修改请求必须满足同源边界；HTTPS 部署必须启用 Secure Cookie。

### owner 与权限

- 每条新链接必须具有 `owner_id`。
- 普通用户只能列出、读取、修改和查看自己链接的统计。
- 对象不存在和跨用户访问返回相同 `404`，避免短码枚举泄露。
- 公共 `GET /s/{code}` 不要求登录。
- v1.9 历史链接归属禁用的 `legacy-system`，普通用户不能自动认领。
- v2.0 不实现平台管理员、跨用户管理或 RBAC。

### 创建与生命周期

- 创建请求接受 HTTP(S) 原始 URL、可选未来过期时间和可选自定义短码。
- 未指定自定义短码时继续使用持久化 id 生成 Base62 短码。
- 自定义短码为 4-32 个 ASCII 字母、数字、`_` 或 `-`，大小写敏感且全局唯一。
- 保留入口相关短码，数据库唯一约束裁决并发冲突。
- 链接状态为 `active` 或 `disabled`；过期由可空 UTC `expires_at` 动态计算。
- 用户可禁用、恢复和修改过期时间，不提供硬删除或链接转移。
- 不存在、禁用和过期链接对公共访问统一返回 `404`。

### 管理页面和契约

- `/app/` 同源页面覆盖注册 / 登录、创建、分页列表、筛选、禁用 / 恢复、过期时间、统计和错误反馈。
- 页面不依赖 React、Vue 或 Node.js 构建链。
- `docs/openapi.yaml` 覆盖实际公开路由，并可由 Compose 入口获取。
- “管理”指普通用户管理自己的链接，不是平台运营后台。

### 持久化与迁移

- MySQL 是用户、会话、链接 owner 和统计的事实存储。
- Redis 只作可选查询缓存和可选全局创建限流，故障不改变事实语义。
- 空库和 v1.9 数据库使用相同迁移入口升级到 005。
- 历史链接、生命周期和统计外键在升级中保持可用。
- 重复迁移可安全检测；回滚 005 必须显式确认用户与会话数据丢失。

### 兼容与可靠性

- 现有健康检查、Prometheus / Grafana、Redis 降级和创建限流继续工作。
- Kafka producer 保持异步 fail-open，不影响跳转和 readiness。
- 访问统计继续使用 `event_id` 幂等、重试、DLQ、lag、受控重放和 retention 边界。
- `HttpServer/` 不引入身份、owner 或页面业务类型。

## v2.0 验收条件

- 两个用户之间的列表、详情、更新和统计水平越权全部失败且不泄露对象存在性。
- 重复用户名和重复自定义短码在并发下只有一个请求成功。
- 原始密码和会话 token 不出现在数据库摘要之外的持久化、错误、日志或指标中。
- 注册、登录、退出、过期、撤销和账号禁用行为可重复验证。
- 管理页面可从用户入口完成创建、跳转、生命周期和统计闭环。
- 空库、v1.9 升级、回填、重复迁移和回滚均有自动化证据。
- v1.9 API、缓存、限流、健康、监控、Kafka 和统计故障回归继续通过。

执行证据见 [v2.0 验收记录](V2_ACCEPTANCE.md)。

## 非目标

- 平台管理员、用户封禁后台、链接投诉治理和跨用户运营能力。
- 邮箱、验证码、密码找回、OIDC、邀请、计费、配额或自定义域名。
- 复杂 URL 安全扫描、域名黑名单或目标站点到达确认。
- Kubernetes 多副本与发布回滚；属于 v2.1。
- Transactional Outbox 和生命周期审计；属于 v2.2。
- 生产级有状态集群、SLA 或容量承诺。
