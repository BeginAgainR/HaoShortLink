# 短链接需求

状态：草案
当前实现：尚未实现

## V1 目标

ShortLink V1 的目标是实现短链接服务的最小闭环：

- 服务健康检查。
- 创建短链接。
- 根据短码跳转到原始 URL。

## 核心概念

短链接服务维护一组映射关系：

```text
短码 -> 原始 URL
```

用户访问短链接时，服务根据短码找到原始 URL，并返回 HTTP 重定向响应。

## V1 能力

### 健康检查

用于确认服务进程和 HTTP 路由是否可访问。

暂定接口：

```text
GET /api/health
```

### 创建短链接

客户端提交原始 URL，服务生成短码并返回短链接信息。

暂定接口：

```text
POST /api/short-links
```

### 短码跳转

用户访问短码路径，服务查找原始 URL，并返回重定向响应。

暂定接口：

```text
GET /s/{code}
```

默认跳转状态码：

```text
302 Found
```

## 暂不包含

- 用户系统。
- 链接列表查询。
- 链接删除或更新。
- 访问统计。
- 过期策略。
- 限流和风控。
- MySQL 持久化。
- Redis 缓存。
